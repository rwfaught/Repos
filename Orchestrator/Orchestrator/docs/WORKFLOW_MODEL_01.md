# Workflow Model 01 — Bootstrap Relay to Internal Orchestration

## Status

This is a workflow/governance document.
It is not a phase boundary and not implementation authorization.

## Current Bootstrap Relay

Current operating loop:
- ChatGPT Orchestrator performs orientation, ranking, synthesis, boundary drafting, and audit interpretation.
- The human owner/operator is the approval membrane and final authority.
- Codex receives bounded worker packets and performs only the authorized task.
- Repo truth outranks conversation memory.
- Worker reports return to the Orchestrator/human loop for review.

## Role Boundaries

- Human owner/operator: approval authority, priority control, final judgment, system authorship.
- Orchestrator session: orientation, ranking, decision framing, packet drafting, audit synthesis.
- Codex worker: bounded execution/documentation worker under explicit packet constraints.
- Repo documents/state: durable truth source and re-entry anchor.
- Future internal Orchestrator system: intended internalization target for durable workflow discipline.

## Migration Direction

The externally performed method should gradually become repo-backed system behavior, but only through explicit phases/fixes.

Examples of migration targets:
- re-entry stack reading
- authority classification
- ranking pass
- worker packet generation
- audit packet generation
- action log updates
- next-best-move recommendations
- case-state carry-forward

## Non-Goals

- no autonomous self-modification
- no hidden planning
- no channel/runtime implementation authorization
- no broad plugin/configuration platform
- no replacement of human approval
- no Phase 61 authorization
- no treating OpenClaw/channel access as the product center

## Active Product Proof

Current live product proof target:
claims/disputes/appeals casework as bounded, inspectable, local case-bearing machinery.

This target should not erase the existing current success criterion, but it should guide Alpha 3.0 ranking.

## Operating Principle

First make the current human-mediated workflow explicit, then migrate durable parts into repo-backed, inspectable system behavior in bounded increments.
