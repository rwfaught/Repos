# RERANK_01.md

## Re-rank 01: Post-Method-2.0 Forward Work Selection

---

## Goal

Perform the first formal re-rank after the integration of Method 2.0 into the live governance stack.

This artifact exists to determine the true next best move now that the project has:

- a ratified proving-ground strategy
- a ratified proving-ground threshold model
- a ratified proving-ground case-state model
- repo-truth and re-entry discipline integrated
- Method 2.0 live in `ORCHESTRATOR_METHOD.md`

This is a **governance/ranking artifact**.

It is not:
- a feature phase
- a design artifact
- a product-strategy memo
- a code implementation task

Its purpose is narrower:

to identify which layer now legitimately governs forward motion and what the next bounded work packet should be.

---

## Why This Artifact Exists

The project has recently completed a large amount of work in strategy, design, recovery, and governance.

That was necessary.

But Method 2.0 now makes one thing explicit:

**coherent artifact production does not by itself justify the next artifact.**

The project must now stop and ask, under live method discipline:

- what is the active layer?
- what is the active decision membrane?
- what has actually been unlocked?
- what has not yet been unlocked?
- does forward motion now require:
  - another design artifact,
  - a bounded implementation phase,
  - further recovery,
  - or an explicit pause?

This artifact exists to answer that question cleanly.

## Required Method 2.0 Application

This re-rank must explicitly apply the live Method 2.0 rules.

At minimum, the worker must identify:

1. **Active layer**
   - which layer currently governs ranking:
     - recovery
     - strategy
     - design
     - implementation
     - validation

2. **Active decision membrane**
   - the current live filter through which work should be judged

3. **Live ranking docs**
   - which docs currently outrank normal feature motion

4. **Live design constraints**
   - which proving-ground design artifacts currently constrain forward work

5. **Decision-unlocked analysis**
   - what the ratified strategy/design stack has unlocked
   - what it has not yet unlocked

6. **Design-stack stop analysis**
   - whether another design artifact is justified
   - or whether design momentum should stop here

---

## Core Ranking Question

Given the current proving-ground stack and the live Method 2.0 rules:

**What is the next bounded work packet that is most justified now?**

This question must be answered against the current proving ground:

**claims / disputes / appeals-style casework for consequential personal or small-business paperwork problems**

and against the current engine identity:

**local bounded-trust orchestration for messy, consequential, resumable workflows where raw AI is useful but operationally unreliable.**

---

## Candidate Outcome Types

The re-rank may conclude that the NBM is one of the following:

### Outcome A: Another design artifact
Only if a clear unresolved design gap remains and Method 2.0 design-stack stop discipline still permits it.

### Outcome B: First bounded implementation phase
If the proving-ground stack is now strong enough that the next justified move is implementation.

### Outcome C: Validation / reconciliation step
If current repo truth or phase truth still blocks forward motion.

### Outcome D: Explicit pause
If the project is not yet in a condition to move honestly.

The worker must choose one.

---

## Required Ranking Standard

The recommended next move must be justified in terms of all of the following:

- proving-ground relevance
- engine relevance
- repeated-use viability
- friction-budget discipline
- current design-stack sufficiency
- repo-truth discipline
- whether the move is unlocked by the current stack rather than merely adjacent to it

If the recommendation cannot be justified on those grounds, it is too weak.

---

## What This Artifact Is NOT About

This artifact is NOT about:

- resuming work by enthusiasm alone
- producing another design artifact because it sounds coherent
- broadly auditing the whole repo
- inventing a roadmap
- reopening already-ratified strategic decisions without cause
- treating all docs as equally active

This is a bounded ranking decision only.
